FORMAT DOCC REQUIS:

Pour les types (struct, class, enum, protocol):
```swift
/// Description courte sur une ligne.
///
/// Description detaillee sur plusieurs lignes si necessaire.
/// Peut inclure du contexte, des cas d'usage, etc.
///
/// ## Topics
///
/// ### Creating an Instance
/// - ``init()``
///
/// ### Properties
/// - ``propertyName``
```

Pour les fonctions/methodes:
```swift
/// Description de ce que fait la fonction.
///
/// Description detaillee avec contexte si necessaire.
///
/// ```swift
/// // Exemple d'utilisation
/// let result = myFunction(param: value)
/// ```
///
/// - Parameters:
///   - param1: Description du premier parametre.
///   - param2: Description du second parametre.
///
/// - Returns: Description de la valeur retournee.
///
/// - Throws: `ErrorType` si condition d'erreur.
///
/// - Note: Information supplementaire importante.
/// - Warning: Avertissement si necessaire.
/// - Important: Point critique a ne pas ignorer.
```

Pour les proprietes:
```swift
/// Description de la propriete.
///
/// Valeur par defaut: `valeur` si applicable.
```

REGLES:
1. Documente TOUS les elements publics
2. Premiere ligne = resume concis
3. Exemples de code quand pertinent
4. Utilise les callouts DocC (Note, Warning, Important, Tip)
5. References croisees avec ``Symbol``
6. Groupes Topics pour organisation
